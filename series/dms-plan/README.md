# DMS Plan

*A presentation and handout series for data management, workflow design, and repository delivery*

This series is for materials focused on the real work that sits between local project systems and successful repository delivery.

Its central question is:

> how does a Data Management and Sharing Plan become a working path from source systems to structured, documented, validated, shareable data?

These materials are not primarily Iowa outreach artifacts.

They are operational DMS materials that can support research teams, labs, institutional conversations, and repository-delivery planning wherever that work is happening.

## Current entries

- **Data Management and Sharing Plan** — the operational workflow deck explaining the messy middle from project systems to repository delivery

## Series role

This series should answer practical DMS questions:

- how to think about a DMS Plan as a workflow rather than a form
- where the messy middle lives between collection systems and repository submission
- what kinds of technical, organizational, and documentation work make the plan real
- where Steve Slevinski adds practical value in planning and execution

## Recommended order of use

The strongest default order is:

1. **Data Management and Sharing Plan presentation** for the main workflow argument
2. **Data Management and Sharing Plan handout** for a careful written version

## Relationship to the other series

This series stands alongside, rather than inside, the Iowa, science, and infrastructure series.

The relationship is best understood as:

1. **DMS Plan** for operational workflow and repository-delivery work
2. **University of Iowa** for institution-specific outreach, PI placement, and collaboration structure
3. **The Science of Sutton SignWriting** for scientific framing and research development
4. **Sutton SignWriting Infrastructure Initiative** for long-term stewardship, partnerships, and durable support

That keeps DMS work visible as its own lane while still allowing it to support other conversations when relevant.
