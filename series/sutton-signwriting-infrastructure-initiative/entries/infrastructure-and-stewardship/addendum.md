# Infrastructure Snapshot

_Companion addendum for **Infrastructure and Stewardship**_

## Purpose

This one-page snapshot is designed for fast reading.

It distills the larger handout into a compact institutional view of:

* what already exists
* what is currently being stewarded
* where the main risks sit
* what a host institution would actually be taking on

The goal is to help a PI, administrator, or collaborator understand in a few minutes that Sutton SignWriting is already operating as infrastructure.

## Snapshot Summary

Sutton SignWriting is not a greenfield initiative.

It already exists as a coordinated infrastructure across:

* domains and public websites
* servers and platform systems
* software and code repositories
* datasets and corpus assets
* standards and encoding systems

The central institutional task is not invention.

It is:

> **stabilization, documentation, and institutionalization**

## Asset Classes

### 1. Domains and Public Web Presence

Current stewardship includes a portfolio of domains such as:

* SignWriting
* SignPuddle
* SignBank
* Sutton SignWriting
* Valerie Sutton
* related archival, project, and personal domains

These domains support:

* public access
* application entry points
* documentation and standards
* archives and historical continuity

### 2. Servers and Platform Systems

The ecosystem depends on:

* legacy systems still carrying historical continuity
* active production systems
* static and dynamic hosting
* ongoing redevelopment through Platform V3

These systems support:

* websites
* dictionaries
* downloads
* developer resources

### 3. Software and Code

The software environment includes:

* JavaScript, Python, and PHP libraries
* rendering and font systems
* clients, viewers, and components
* APIs and platform code

This is a mature technical base with open-source and production-facing value.

### 4. Data and Corpora

Key data assets include:

* SignPuddle dictionaries
* SPML files
* millions of sign entries
* lexical and textual materials
* educational and religious literature collections

This is a rare corpus of written signed language with research and preservation value.

### 5. Standards and Encoding

Core standards infrastructure includes:

* ISWA 2010
* Formal SignWriting in ASCII (FSW)
* SignWriting in Unicode (SWU)
* associated fonts and rendering systems

Together these form a stable machine-readable framework for SignWriting.

## Stewardship Chain

Current stewardship already follows a clear lineage:

* Valerie Sutton: originator and foundational system design
* Steve Slevinski: technical steward, encoding architect, and maintainer
* next-generation continuity path: identified but still too informal

This means continuity exists, but it is not yet institutionally strong enough for the scale of the assets involved.

## Immediate Risk Areas

The main risks are structural:

* concentrated operational knowledge
* dependence on very few maintainers
* incomplete documentation
* limited redundancy across systems and responsibilities
* preservation risk for domains, data, and services

These are typical signs of infrastructure that has grown faster than its institutional support.

## What a Host Institution Would Be Doing

A host institution would not be:

* funding an idea from zero
* being asked to invent relevance

A host institution would be:

* stabilizing existing infrastructure
* improving documentation and continuity
* supporting preservation and modernization
* making grants, research use, and collaboration more credible

## Institutional Reading

The clearest institutional conclusion is this:

> Sutton SignWriting has already crossed the line from interesting project to real infrastructure.

The remaining question is whether that infrastructure will remain dependent on an informal stewardship model or receive the durable academic and technical home its current value now warrants.

## Bottom Line

Already in operation:

* domains
* servers
* software
* datasets
* standards

Now needed:

* stabilization
* documentation
* institutionalization
