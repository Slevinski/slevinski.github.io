# Other Sign Language Scripts and Their Conceptions Comparison Notes

**Other Sign Language Scripts and Their Conceptions**  
ASL-phabet, HamNoSys, ASLwrite, and what they imply about written sign language

These comparison notes record a compact comparison that can support institutional or collaborator conversations without turning the larger initiative into a script-comparison project.

## Compact comparison

**ASL-phabet**

- **Main conception**: linear alphabetic script
- **Institutional setting**: university-connected educational setting
- **Support / funding pattern**: school and academic collaboration
- **Vision**: establish an alphabetic principle for ASL and support a bridge toward written English

**HamNoSys**

- **Main conception**: analytic notation
- **Institutional setting**: academic research environment
- **Support / funding pattern**: scholarly and institutional support for transcription work
- **Vision**: provide a detailed cross-linguistic notation for sign representation

**ASLwrite**

- **Main conception**: plane-based script with early partial encoding experiments
- **Institutional setting**: outside formal academia
- **Support / funding pattern**: community-driven support
- **Vision**: sustain alternate spatial script design and handwriting culture

**SignWriting**

- **Main conception**: plane-based writing plus technical infrastructure
- **Institutional setting**: internationally used but still under-housed institutionally
- **Support / funding pattern**: independent stewardship plus growing need for durable institutional backing
- **Vision**: function as real writing, technical infrastructure, and long-term public continuity

## Different communities, different pressures

One reason these systems should not be flattened into a simple list is that they emerged from different communities and different design pressures.

- **ASL-phabet** is easiest to understand through educational and alphabetic framing
- **HamNoSys** is easiest to understand through linguistic notation and representation
- **ASLwrite** is easiest to understand through alternate script design and handwriting culture
- **SignWriting** increasingly has to be understood through both script culture and infrastructure stewardship

This is why the systems can coexist in the same conversation without all doing the same job.

## Why institutional setting matters

Institutional setting changes what a system can realistically become.

- a system born in a school-and-university collaboration may stay closely tied to instructional goals
- a system born in academic notation research may remain strongest in transcription and analysis
- a system sustained by community adoption may prioritize everyday script culture and handwriting
- a system used publicly across countries, software, and standards eventually becomes an infrastructure and stewardship problem

This is one reason SignWriting cannot be reduced to a simple script comparison.

## Useful distinctions

The following distinctions help keep the conversation disciplined:

- **script** is not the same as **notation**
- **plane-based visible writing** is not the same as **linear encoded storage**
- **script code** is not the same as **encoding name**
- **educational use** is not the same as **shared public infrastructure**
- **handwriting tradition** is not the same as **software or standards stewardship**
- **word-centered instructional use** is not the same as a broad public writing ecosystem

One additional distinction matters in discussions of SignWriting:

- **formalized writing system** is not the same as **illustrative drawing**

These distinctions matter because many sign-language discussions collapse them.

## Standards note on `Sgnw`

`Sgnw` is the ISO 15924 script code for SignWriting.

In BCP 47 language tags, a form such as `ase-Sgnw` identifies American Sign Language written in the SignWriting script.

That is a script-level statement.

It is not the same thing as naming the concrete encoding used to serialize the text.

For the standards stack in this project:

- **Formal SignWriting** names the public computable model
- **FSW** and **SWU** name the two canonical encodings of that model
- `Sgnw` remains the script identifier used in language tags

This is the clean way to say that SignWriting has a standards position stronger than the other script conceptions named in this presentation without confusing script identity with encoding format.

## How to speak about ASL-phabet

The most useful tone is:

- visible enough to acknowledge
- not central to the SignWriting agenda
- one current answer to the question of how signed writing should be structured

Avoid letting it become the center of the infrastructure case.

## How to speak about HamNoSys

The most useful tone is:

- important notation name
- valuable for analytic representation
- different in purpose from an ordinary writable script

This makes room for respect without confusion.

## How to speak about ASLwrite

The most useful tone is:

- another serious plane-based script
- helpful for keeping handwriting and spatial writing alive as live design questions
- fair about the current technical limits: partial coverage, hand-built strings, and no mature editor or implementation path yet

This allows a friendly and accurate acknowledgment.

## Handwriting note

The handwriting question deserves to remain open and constructive.

SignWriting already has handwriting traditions:

- shorthand
- cursive
- block printing

These traditions have never reached uniform mass adoption, but they remain real and can grow further.

ASLwrite is worth acknowledging partly because it reminds the field that handwriting culture is not a side issue.

## Bottom line

This initiative does not need to deny the existence of other systems.

It only needs to keep its priorities clear:

- acknowledge the landscape
- avoid reactive comparison as the central project
- build SignWriting forward as durable writing and durable infrastructure

That depends in part on speaking clearly about what SignWriting already is: a formalized writing system with defined encodings, grammar, search, rendering, and implementation pathways, not merely a set of sign illustrations.
